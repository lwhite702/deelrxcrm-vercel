# Create Knowledge Base Article

Create a new knowledge base article with content, metadata, and publication settings.

<Param body="title" type="string" required>
Article title (maximum 200 characters)
</Param>

<Param body="content" type="string" required>
Article content in Markdown format
</Param>

<Param body="excerpt" type="string">
Brief summary or excerpt (maximum 500 characters)
</Param>

<Param body="category" type="string" required>
Category identifier for the article
</Param>

<Param body="tags" type="array">
Array of tag strings for categorization
</Param>

<Param body="status" type="string">
Publication status (draft, published) - default: draft
</Param>

<Param body="slug" type="string">
URL-friendly slug (auto-generated if not provided)
</Param>

<Param body="publishAt" type="string">
Schedule publication for future date (ISO 8601)
</Param>

## Request

<CodeGroup>

## Request Fields

<Param body="title" type="string" required>
Article title (3-200 characters)
</Param>

<Param body="content" type="string" required>
Article content in Markdown format (minimum 50 characters)
</Param>

<Param body="excerpt" type="string">
Brief summary displayed in article lists (maximum 500 characters)
</Param>

<Param body="category" type="string" required>
Category identifier - must be an existing category
</Param>

<Param body="tags" type="array">
Array of tag strings (maximum 20 tags, each up to 50 characters)
</Param>

<Param body="status" type="string">
Publication status:
- `draft` - Article saved but not published (default)
- `published` - Article is live and visible to users
</Param>

<Param body="slug" type="string">
URL-friendly identifier (auto-generated from title if not provided)
Must be unique, lowercase, and contain only letters, numbers, and hyphens
</Param>

<Param body="publishAt" type="string">
Schedule article publication for a future date (ISO 8601 format)
Only valid when status is "draft"
</Param>

## Validation Rules

### Title Requirements
- Length: 3-200 characters
- Must be unique within the category
- Cannot contain only whitespace

### Content Requirements
- Minimum length: 50 characters
- Supports Markdown formatting
- Maximum size: 1MB

### Slug Requirements
- Must be unique across all articles
- Contains only lowercase letters, numbers, and hyphens
- Length: 3-100 characters
- Auto-generated from title if not provided

### Category Validation
- Must reference an existing category
- User must have permission to create articles in the category

### Tags Validation
- Maximum 20 tags per article
- Each tag: 1-50 characters
- Tags are case-insensitive and automatically normalized

## Automatic Features

### Auto-Generated Fields
- **Slug**: Generated from title if not provided
- **Excerpt**: Generated from first 200 characters of content if not provided
- **Author**: Set to current authenticated user
- **Timestamps**: Created and updated timestamps set automatically

### Content Processing
- Markdown is validated and sanitized
- Links are checked for validity
- Images are processed and optimized
- Table of contents generated automatically

## Publication Behavior

### Draft Articles
- Saved but not visible to end users
- Can be edited and updated freely
- Can be scheduled for future publication

### Published Articles
- Immediately visible to users with appropriate permissions
- Indexed for search functionality
- Generate activity notifications
- Version history maintained

## Error Codes

- `VALIDATION_ERROR` - Invalid input data
- `CATEGORY_NOT_FOUND` - Specified category does not exist
- `SLUG_ALREADY_EXISTS` - Provided slug is already in use
- `CONTENT_TOO_LARGE` - Article content exceeds maximum size
- `INSUFFICIENT_PERMISSIONS` - User lacks permission to create articles
- `RATE_LIMIT_EXCEEDED` - Too many requests

## Best Practices

### Content Guidelines
- Use clear, descriptive titles
- Structure content with headers and sections
- Include relevant tags for discoverability
- Write concise, informative excerpts

### SEO Optimization
- Use descriptive slugs
- Include relevant keywords in title and content
- Structure content with proper heading hierarchy
- Add internal links to related articles

### Maintenance
- Review and update articles regularly
- Use consistent tagging conventions
- Organize content into logical categories
- Monitor article performance and engagement
