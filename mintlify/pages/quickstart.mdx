## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 18+** - [Download Node.js](https://nodejs.org/)
- **PostgreSQL Database** - We recommend [Neon](https://neon.tech/) for managed PostgreSQL
- **Git** - [Download Git](https://git-scm.com/)

## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/lwhite702/deelrxcrm-vercel.git
    cd deelrxcrm-vercel
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    # or
    yarn install
    # or 
    pnpm install
    ```
  </Step>

  <Step title="Environment Setup">
    Create a `.env.local` file in the root directory:
    
    ```bash
    cp .env.example .env.local
    ```
    
    Update the environment variables:
    
    ```env .env.local
    # Database
    DATABASE_URL="postgresql://username:password@localhost:5432/deelrxcrm?sslmode=require"
    
    # Authentication (Clerk)
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."
    CLERK_SECRET_KEY="sk_test_..."
    
    # Payments (Stripe)
    STRIPE_SECRET_KEY="sk_test_..."
    STRIPE_WEBHOOK_SECRET="whsec_..."
    
    # CORS Origins (comma-separated)
    CORS_ORIGIN="http://localhost:3000"
    ```
  </Step>

  <Step title="Database Setup">
    Run the database migrations:
    
    ```bash
    npm run db:push
    ```
    
    This will create all the necessary tables and relationships in your PostgreSQL database.
  </Step>

  <Step title="Start Development Server">
    ```bash
    npm run dev
    ```
    
    Your application will be available at [http://localhost:3000](http://localhost:3000)
  </Step>
</Steps>

## Verification

Once your development server is running, verify the installation:

<AccordionGroup>
  <Accordion title="Check Database Connection">
    Open Drizzle Studio to explore your database:
    
    ```bash
    npm run db:studio
    ```
    
    This opens a web interface at `http://localhost:4983` where you can view your database tables.
  </Accordion>

  <Accordion title="Test API Endpoints">
    Test a few key endpoints to ensure everything is working:
    
    ```bash
    # Health check
    curl http://localhost:3000/api/health
    
    # Teams endpoint (requires authentication)
    curl http://localhost:3000/api/teams
    ```
  </Accordion>

  <Accordion title="Access the Dashboard">
    Navigate to [http://localhost:3000](http://localhost:3000) and you should see the DeelRx CRM login page.
    
    Sign up for a new account or sign in with existing credentials.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database Connection Issues">
    If you encounter database connection issues:
    
    1. Verify your `DATABASE_URL` is correct
    2. Ensure your PostgreSQL server is running
    3. Check that the database exists and is accessible
    4. For Neon, make sure you include `?sslmode=require` in the URL
  </Accordion>

  <Accordion title="Authentication Problems">
    For Clerk authentication issues:
    
    1. Verify your Clerk keys in `.env.local`
    2. Check that your domain is configured in Clerk dashboard
    3. Ensure CORS origins are properly set
  </Accordion>

  <Accordion title="Build Errors">
    If you encounter build errors:
    
    1. Clear node_modules and reinstall dependencies
    2. Check for TypeScript errors: `npm run type-check`
    3. Verify all environment variables are set
    4. Run `npm run build` to check for build issues
  </Accordion>
</AccordionGroup>
